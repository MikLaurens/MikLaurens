# Документация и Руководство пользователя  
Данное программное приложение предназначено для автоматизации учёта производства и продаж малого предприятия (ООО «Мясной дом»). Ниже представлена подробная документация, описывающая структуру, функциональность и порядок использования приложения.

---

## 1. Общая информация

### 1.1 Цель приложения  
Главная задача приложения — упростить и ускорить процессы управления запасами и продажами на предприятии, а также обеспечить удобные инструменты для формирования отчётов.  

В результате использования приложения:  
1. Автоматизируется учёт остатков продукции и сырья.  
2. Упрощается процесс регистрации продаж и производства.  
3. Предоставляются различные виды отчётов (продажи, складские остатки, производство).  

### 1.2 Ключевые задачи  
- Управление товарами (мясная продукция и ингредиенты).  
- Учёт производства: регистрация производственных партий, списание сырья, увеличение остатков готовой продукции.  
- Учёт продаж: автоматическое обновление остатков и подсчёт выручки за выбранный период.  
- Генерация отчётов для контроля бизнеса.

---

## 2. Установка и системные требования

### 2.1 Требования к окружению  
1. Python 3.7 или выше.  
2. Библиотека PyQt5 (для графического интерфейса).  
3. SQLite3 (входит в стандартную поставку большинства Python-сред).

### 2.2 Установка  
1. Установите Python (если ещё не установлен).  
2. Убедитесь, что PyQt5 доступен в вашем окружении:
   ```bash
   pip install PyQt5
   ```
3. Склонируйте репозиторий с приложением (командой `git clone <URL>` или загрузите архив).  
4. (Опционально) Создайте виртуальное окружение и активируйте его:
   ```bash
   python -m venv venv
   source venv/bin/activate  # для Linux / MacOS
   venv\Scripts\activate     # для Windows
   ```
5. Установите любые дополнительные зависимости, указанные в файле `requirements.txt` (если он присутствует):
   ```bash
   pip install -r requirements.txt
   ```

---

## 3. Структура базы данных

Приложение использует базу данных SQLite3. При первом запуске (или при ручной инициализации) создаются следующие таблицы:

1. **products**  
   - `id` (INTEGER, PRIMARY KEY)  
   - `name` (TEXT) — наименование товара/продукта  
   - `price` (REAL) — цена за единицу  
   - `stock` (INTEGER) — текущее количество на складе  

2. **raw_materials**  
   - `id` (INTEGER, PRIMARY KEY)  
   - `name` (TEXT) — наименование сырья  
   - `quantity` (INTEGER) — количество сырья  

3. **sales**  
   - `id` (INTEGER, PRIMARY KEY)  
   - `product_id` (INTEGER, FOREIGN KEY) — ссылка на таблицу `products`  
   - `quantity` (INTEGER) — количество проданного товара  
   - `sale_date` (DATE) — дата продажи  

4. **production**  
   - `id` (INTEGER, PRIMARY KEY)  
   - `product_id` (INTEGER, FOREIGN KEY) — ссылка на таблицу `products`  
   - `quantity` (INTEGER) — количество произведённого товара  
   - `production_date` (DATE) — дата производства  

---

## 4. Функциональные возможности

### 4.1 Управление товарами
- Добавление нового товара (мясного изделия) с полями: название, цена, количество на складе.  
- Редактирование существующего товара.  
- Удаление товара из базы данных (с учётом возможных ограничений логирования исторических данных).

### 4.2 Учёт производства
- Регистрация производственной партии (выбор товара, количество, дата производства).  
- Списание используемого сырья (если ведётся учёт через модуль сырьевых материалов).  
- Увеличение остатка готовой продукции на складе.

### 4.3 Учёт продаж
- Регистрация продаж (выбор товара, количество, дата).  
- Автоматический расчёт выручки = цена × количество.  
- Обновление остатков товаров.

### 4.4 Генерация отчётов
- Выбор периода (начальная и конечная дата) и формирование отчёта по продажам (суммарная выручка за период).  
- Вывод текущих остатков на складе с детализацией по каждому товару (включая либо отдельно, либо в совокупности с сырьём).  
- Отчёт по производству с детализацией: когда и сколько было произведено.

## 5. Сценарии использования

### 5.1 Типичный процесс учёта производства
1. Заведите или отредактируйте новые товары (если они ещё не добавлены).  
2. В разделе «Производство» введите данные о произведённой партии.  
3. Приложение увеличит количество произведённых товаров в базе.  

Если используется учёт сырья:  
1. Введите склады сырьевых продуктов (например, мясо, специи).  
2. При производстве часть сырья будет списываться, и вы сможете видеть текущее количество доступных ингредиентов.

### 5.2 Типичный процесс учёта продаж
1. Выберите в интерфейсе «Продажи» нужный товар.  
2. Введите количество и дату продажи.  
3. Приложение автоматически уменьшит остаток товара и создаст запись в таблице `sales`.  
4. После нескольких продаж можно сформировать отчёт по продажам за интересующий период.

### 5.3 Формирование отчётов
1. Перейдите в раздел «Отчёты».  
2. Укажите период (например, c «2023-01-01» по «2023-12-31») для отчёта по продажам.  
3. Нажмите «Сформировать», после чего вы увидите общую информацию о количестве проданных товаров, суммарную выручку и, при необходимости, детализацию.  
4. Для отчёта об остатках печатается список товаров/сырья с текущим доступным количеством.  
5. Для отчёта по производству можно отфильтровать записи по дате, чтобы увидеть количество выпущенной продукции.

## 6. Руководство пользователя

### 6.1 Запуск приложения
1. Перейдите в папку с проектом.  
2. (Убедитесь, что активировано виртуальное окружение, если оно используется.)  
3. Запустите главный файл (например, `main.py`):
   ```bash
   python main.py
   ```
4. На экране появится главное окно приложения с кнопками/вкладками (зависит от реализации в PyQt5).

### 6.2 Основные разделы интерфейса
- **Управление товарами (Products)**:  
  В этом разделе можно добавлять, редактировать и удалять товары.  
- **Учёт производства (Production)**:  
  Раздел для фиксации факта производства определённой продукции, с указанием количества и даты.  
- **Учёт продаж (Sales)**:  
  Окно для ввода информации о проданных товарах, количестве и дате продажи.  
- **Отчёты (Reports)**:  
  Интерфейс для создания отчётов за нужный период, а также получения сводных данных по складу и производству.

### 6.3 Пример использования
1. **Добавить новый продукт**:  
   - Зайдите в раздел «Products».  
   - Нажмите «Добавить продукт».  
   - Введите название (например, «Колбаса Докторская»), цену (например, 250 руб.), и стартовое количество (например, 100).  
   - Сохраните изменения.

2. **Произвести партию**:  
   - Перейдите в «Production».  
   - Выберите «Колбаса Докторская», укажите количество (например, 50), дату производства (текущая дата или другая).  
   - Сохраните. Приложение прибавит 50 к текущему остатку «Колбаса Докторская».

3. **Продать товар**:  
   - В разделе «Sales» выберите «Колбаса Докторская».  
   - Введите количество (например, 10), дату продажи.  
   - Сохраните. Остаток станет на 10 меньше, а в таблицу `sales` добавится соответствующая запись.

4. **Сформировать отчёт по продажам**:  
   - Зайдите в «Reports».  
   - Укажите диапазон дат (например, c 01.01.2023 по 31.01.2023).  
   - Нажмите «Формировать отчёт». Отобразится список продаж и итоговая выручка за этот период.

## 7. Тестирование

### 7.1 Функциональное тестирование
Проверьте каждую функцию:
- Добавление/редактирование/удаление товара.  
- Корректность списания/прибавления остатков при продаже и производстве.  
- Формирование отчётов за разные даты.  

### 7.2 Нагрузочное тестирование
- Заполните базу данных большим количеством записей (сотни/тысячи).  
- Убедитесь, что время ответа приложения (запросы к базе) не превышает 2 секунд.

### 7.3 Безопасность
- Параметризованные запросы в SQLite: убедитесь, что пользовательский ввод не встраивается напрямую в SQL.  
- Резервное копирование базы данных (копирование файла .db) и проверка его восстановления.
